# テスト設計書(test-design.md)

## 対象範囲

社内APIキー発行台帳アプリ ステップ1「発行登録フォーム+入力検証」。
情報源は [requirements.md](./requirements.md) および [screen-spec.md](./screen-spec.md) のみとする(アプリのソースコードは参照しない)。

## 共通前提

すべてのテストケースは、個別に断りがない限り以下を前提とする。

- P1: アプリを `python -m app --test` で起動した直後である(一時ファイルSQLite使用、ウィンドウ位置(100,100)・サイズ800×600)。
- P2: 環境変数 `APP_FAKE_NOW` に `2026-07-09T12:00:00` を設定して起動する。
- P3: コントロールは objectName([screen-spec.md](./screen-spec.md) 参照)で特定する: `userNameLineEdit` / `departmentComboBox` / `budgetLineEdit` / `issueButton` / `statusLabel`。
- P4: 「入力する」は該当欄へ値を設定する操作、「[発行]をクリック」は `issueButton` の押下操作を指す。

## 判定上の注意

- ステータス欄(`statusLabel`)の期待文言は requirements.md「エラーメッセージ一覧」との**完全一致**で判定する(部分一致・正規化比較は不可。ただしMSG-006の `<キー>` 部は正規表現 `^sk-[a-z0-9]{32}$` に合致する文字列)。
- 「保存されないこと」(REQ-016)のDB内部検証は、ステップ1のUIからは直接観測できない。本書では期待結果として記載し、検証手段(一時DBファイルの直接参照等)はテスト実装時に定める。UIから観測可能な代替判定(ステータス欄がエラー文言のままであること)を併用してよい。
- 入力値の表記: `""` は空文字、`"␣"` は半角空白1文字を表す。

## 予算上限の境界値分析

| 入力値 | 分類 | 期待動作 | ケースID |
|---|---|---|---|
| `""`(空) | 異常系(未入力) | MSG-003 | TC-014 |
| `abc`(非数値) | 異常系(形式) | MSG-004 | TC-015 |
| `0` | 境界値(下限-1) | MSG-005 | TC-020 |
| `1` | 境界値(下限) | 発行成功 | TC-021 |
| `1000000` | 境界値(上限) | 発行成功 | TC-022 |
| `1000001` | 境界値(上限+1) | MSG-005 | TC-023 |
