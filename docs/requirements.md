# 要件仕様書(requirements.md)

## 対象範囲

社内APIキー発行台帳アプリのうち、**ステップ1「発行登録フォーム+入力検証」** のみを対象とする。
一覧表示・検索・削除・編集など、発行登録以外の機能は本書の対象外(将来ステップ)とする。

## 記法

各要件は EARS記法(Easy Approach to Requirements Syntax)で記述し、`REQ-001` から始まる連番のIDを付与する。
行頭の `[パターン名]` は当該要件が EARSのどのパターンに該当するかを示す注記であり、要件文そのものではない。

- `[Ubiquitous]` : システムは常に〜する
- `[Event-driven]` : 〜した場合、システムは〜する
- `[Unwanted behavior]` : 〜の場合(異常系)、システムは〜する
- `[State-driven]` : 〜である間、システムは〜する

## 用語定義

| 用語 | 定義 |
|---|---|
| 発行登録フォーム | 利用者名・部署・予算上限を入力し、[発行]ボタンでAPIキーを発行する画面 |
| ステータス欄 | 発行結果またはエラーメッセージを表示する画面上のテキスト表示領域 |
| トリム | 文字列先頭・末尾の半角空白文字(0x20)を取り除く処理。全角空白はトリム対象に含めない |
| 発行日時 | APIキーを発行した日時。環境変数 `APP_FAKE_NOW` (ISO8601形式)が設定されている場合はその値を、設定されていない場合はシステムの現在時刻を用いる |

## 1. 画面初期状態

- REQ-001 [Ubiquitous] システムは、発行登録フォームの表示時、利用者名入力欄を空文字の状態で表示しなければならない。
- REQ-002 [Ubiquitous] システムは、発行登録フォームの表示時、部署コンボボックスに「開発部」「営業部」「管理部」の3項目をこの順序で表示し、初期選択項目を「開発部」としなければならない。
- REQ-003 [Ubiquitous] システムは、発行登録フォームの表示時、予算上限入力欄を空文字の状態で表示しなければならない。
- REQ-004 [Ubiquitous] システムは、発行登録フォームの表示時、ステータス欄を空文字の状態で表示しなければならない。
- REQ-005 [Ubiquitous] システムは、発行登録フォームの表示時、[発行]ボタンを活性状態(クリック可能)で表示しなければならない。
- REQ-006 [State-driven] 部署コンボボックスに選択可能な項目が存在する間、システムは常に3項目(開発部・営業部・管理部)のいずれか1項目を選択済みの状態に保たなければならない。したがって部署コンボボックスの選択状態を原因とする入力検証エラーは発生しない。

## 2. 検証の共通ルール

- REQ-007 [Ubiquitous] システムは、利用者名入力欄および予算上限入力欄の値を検証する前に、それぞれの値に対してトリムを行わなければならない。
- REQ-008 [Ubiquitous] システムは、利用者名の文字数を、トリム後の文字列に対するUnicode文字単位のカウント(全角・半角を区別しない)で判定しなければならない。
- REQ-009 [Event-driven] ユーザーが[発行]ボタンをクリックした場合、システムは以下の順序で検証を行い、最初に条件へ合致した1件のエラーのみをステータス欄に表示し、以降の検証項目を評価してはならない。
  1. 利用者名の未入力チェック(REQ-010)
  2. 利用者名の文字数チェック(REQ-011)
  3. 予算上限の未入力チェック(REQ-012)
  4. 予算上限の整数形式チェック(REQ-013)
  5. 予算上限の範囲チェック(REQ-014)
  6. 上記いずれにも該当しない場合、検証OKとして発行処理(REQ-015〜)を行う

## 3. 利用者名の検証

- REQ-010 [Unwanted behavior] トリム後の利用者名が0文字の場合、システムはステータス欄に文言「利用者名を入力してください」を表示し、SQLiteへの保存を行ってはならない。
- REQ-011 [Unwanted behavior] トリム後の利用者名の文字数が51文字以上の場合、システムはステータス欄に文言「利用者名は50文字以内で入力してください」を表示し、SQLiteへの保存を行ってはならない。
  - 補足(境界値): トリム後50文字は本チェックに合致しない(検証OK側)。トリム後51文字は本チェックに合致する(検証NG側)。

## 4. 予算上限の検証

- REQ-012 [Unwanted behavior] トリム後の予算上限が0文字の場合、システムはステータス欄に文言「予算上限を入力してください」を表示し、SQLiteへの保存を行ってはならない。
- REQ-013 [Unwanted behavior] トリム後の予算上限が半角数字(0-9)のみで構成されていない場合(符号・小数点・カンマ・全角数字・英字等を含む場合を含む)、システムはステータス欄に文言「予算上限は整数で入力してください」を表示し、SQLiteへの保存を行ってはならない。
  - 補足: 本チェックはREQ-012に合致しなかった場合にのみ評価する。
  - 補足: 先頭に `+` または `-` を含む文字列(例: `-5`、`+100`)は半角数字のみで構成されているとみなさず、本チェックに合致する。
- REQ-014 [Unwanted behavior] トリム後の予算上限を整数値に変換した結果が1未満、または1,000,000を超える場合、システムはステータス欄に文言「予算上限は1〜1,000,000の範囲で入力してください」を表示し、SQLiteへの保存を行ってはならない。
  - 補足: 本チェックはREQ-012およびREQ-013に合致しなかった場合にのみ評価する。
  - 補足(境界値): 整数値0は本チェックに合致する(検証NG側)。整数値1は本チェックに合致しない(検証OK側)。整数値1,000,000は本チェックに合致しない(検証OK側)。整数値1,000,001は本チェックに合致する(検証NG側)。
  - 補足: 先頭に0を含む数字列(例: `007`)は整数値7として扱う。

## 5. 検証NG時の共通動作

- REQ-015 [Unwanted behavior] REQ-010・REQ-011・REQ-012・REQ-013・REQ-014のいずれかに合致した場合、システムは利用者名入力欄・部署コンボボックス・予算上限入力欄の表示内容を検証実行前の状態のまま変更してはならない(値をクリアしない)。
- REQ-016 [Unwanted behavior] REQ-010・REQ-011・REQ-012・REQ-013・REQ-014のいずれかに合致した場合、システムはSQLiteデータベースの既存レコードを変更してはならない。

## 6. 検証OK時(発行処理)

- REQ-017 [Event-driven] REQ-009の検証において1〜5のいずれの条件にも合致しなかった場合、システムは正規表現 `^sk-[a-z0-9]{32}$` に合致するAPIキー文字列を生成しなければならない。
- REQ-018 [Unwanted behavior] 生成したAPIキー文字列がSQLiteデータベースに既存する場合、システムは既存レコードと重複しないAPIキー文字列を得られるまでREQ-017のキー生成を繰り返さなければならない。
- REQ-019 [Event-driven] REQ-017で生成したAPIキーが確定した場合、システムはトリム後の利用者名・選択中の部署・整数値に変換した予算上限・生成したAPIキー・発行日時を1件のレコードとしてSQLiteデータベースに保存しなければならない。
- REQ-020 [Event-driven] 同一の利用者名で複数回発行が行われた場合であっても、システムはその都度新しいレコードとして保存し、利用者名の重複をエラーとしてはならない。
- REQ-021 [Event-driven] REQ-019のレコード保存が完了した場合、システムはステータス欄に文言「発行しました: <キー>」(<キー>は生成したAPIキー文字列に置換)を表示しなければならない。
- REQ-022 [Event-driven] REQ-019のレコード保存が完了した場合、システムは利用者名入力欄・部署コンボボックス・予算上限入力欄の表示内容を発行処理実行前の状態のまま変更してはならない(値をクリアしない)。

## エラーメッセージ一覧(確定文言)

表記ゆれ防止のため、ステータス欄に表示する文言は以下の通りとし、実装・テストともに本表の文言と完全一致させること。

| ID | 条件 | 文言(ステータス欄表示テキスト) |
|---|---|---|
| MSG-001 | 利用者名が未入力(REQ-010) | `利用者名を入力してください` |
| MSG-002 | 利用者名が50文字超過(REQ-011) | `利用者名は50文字以内で入力してください` |
| MSG-003 | 予算上限が未入力(REQ-012) | `予算上限を入力してください` |
| MSG-004 | 予算上限が整数形式でない(REQ-013) | `予算上限は整数で入力してください` |
| MSG-005 | 予算上限が範囲外(REQ-014) | `予算上限は1〜1,000,000の範囲で入力してください` |
| MSG-006 | 発行成功(REQ-021) | `発行しました: <キー>` (`<キー>` は生成したAPIキー文字列に置換。例: `発行しました: sk-ab12cd34ef56gh78ij90kl12mn34op56`) |
