## 発見したバグ一覧

### BUG-001: 利用者名がちょうど50文字のとき誤って拒否される

| 項目 | 内容 |
|---|---|
| 発見した試験 | TC-013(境界値50文字)、TC-009(Unicodeカウント・計50文字) |
| 違反したREQ | REQ-011(50文字は検証OK側と規定) |
| 事象 | トリム後50文字の利用者名で[発行]すると、期待の「発行しました: <キー>」ではなく「利用者名は50文字以内で入力してください」が表示され、保存されない |
| 再現手順 | 1. `python -m app --test` で起動 2. 利用者名に「あ」×50 を入力 3. 予算上限に「100」を入力 4. [発行]をクリック |
| 原因 | 文字数チェックの境界値誤り: `len > 50` とすべき箇所が `len >= 50` になっていた(off-by-one) |
| 状態 | 修正済み |

### BUG-002: 予算上限1,000,000(上限値)が誤って範囲外扱いされる

| 項目 | 内容 |
|---|---|
| 発見した試験 | TC-022(境界値・上限) |
| 違反したREQ | REQ-014(1,000,000は検証OK側と規定) |
| 事象 | 予算上限「1000000」で[発行]すると、期待の発行成功ではなく「予算上限は1〜1,000,000の範囲で入力してください」が表示され、保存されない |
| 再現手順 | 1. `python -m app --test` で起動 2. 利用者名に「田中」を入力 3. 予算上限に「1000000」を入力 4. [発行]をクリック |
| 原因 | 範囲チェックの境界値誤り: 上限比較が `<= 1_000_000` とすべき箇所で `< 1_000_000` になっていた |
| 状態 | 修正済み |

### BUG-003: 予算上限未入力エラーの文言が仕様と不一致(表記ゆれ)

| 項目 | 内容 |
|---|---|
| 発見した試験 | TC-014(予算上限の必須チェック) |
| 違反したREQ | REQ-012 / MSG-003(確定文言「予算上限を入力してください」) |
| 事象 | 予算上限を空のまま[発行]すると「予算上限を入力して**下さい**」と表示される(仕様は「〜して**ください**」。ひらがな/漢字の表記ゆれ) |
| 再現手順 | 1. `python -m app --test` で起動 2. 利用者名に「田中」を入力 3. 予算上限は空のまま 4. [発行]をクリック |
| 原因 | エラーメッセージ定数が仕様書の確定文言と不一致 |
| 状態 | 修正済み |
| 備考 | 文言の完全一致判定を採用していたため検出できた。部分一致にしていたら見逃していた |

### BUG-004: 発行成功後に予算上限欄が勝手にクリアされる

| 項目 | 内容 |
|---|---|
| 発見した試験 | TC-032(成功後の入力保持)。二次被害として TC-028・TC-030 も失敗(1回目の発行で予算欄が消えるため、2回目の[発行]が予算未入力エラーになり連続発行できない) |
| 違反したREQ | REQ-022(発行成功後も入力欄の表示内容を変更してはならない) |
| 事象 | 発行成功後、利用者名・部署は保持されるが予算上限欄だけ空になる |
| 再現手順 | 1. `python -m app --test` で起動 2. 利用者名「田中」・予算上限「100」を入力 3. [発行]をクリック 4. 予算上限欄を確認 → 空になっている 5. 続けて[発行]をクリック → 「予算上限を入力してください」エラー |
| 原因 | 発行成功処理の最後に予算上限欄を空文字で上書きするコードが混入していた |
| 状態 | 修正済み |
